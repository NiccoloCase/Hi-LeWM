# Roadmap

This folder is organized by purpose so results, design notes, and analysis do not get mixed together.

## Structure

- `results/`: CSV exports and status snapshots from sweeps, matrix runs, and experiment summaries.
- `architecture/`: implementation notes, migration plans, and documents that explain how the system is structured.
- `analysis/`: reports, diagnostics, experiment interpretation, and next-step planning.
- `references/`: local copies of papers and extracted paper text used by the roadmap docs.
